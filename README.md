# hatspace

## Naming convention
For branch naming:
- your name/purpose of this branch

For class naming:
- Camel name
- prefix: Hs
- class name should describe the purpose of the class

Suffix for class types:
- widget type: View
- screen type: Screen
- bloc type: Bloc
- bloc state: State
- bloc event: Event
- bloc cubit: Cubit
- common service: Service

 File name:
- Snake case (snake_case)

## Good practice:
- Commit message: should be clear and meaningful of the purpose of each commit
- PR: short, below 1000 LoC
